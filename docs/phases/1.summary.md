## Executive Summary

Phase 1 delivers the MVP of **ResiliPath**: a **Resilience Exercise Execution Engine** refactored into a modular, serverless architecture using **GCP, Firebase, and Vercel**. This phase prioritizes free-tier scalability, strict single-concern boundaries, and SOC 2 / ISO 27001 compliance alignment.

### Strategic Focus
- **Single-Module MVP**: Resilience Exercise Execution Engine only.
- **Modular Sub-Phases**: Broken down into 9 discrete logical domains.
- **GCP/Firebase Stack**: Leveraging Firestore, Firebase Auth, Cloud Functions, and Firebase Storage for cost-effective scaling.
- **Vercel Frontend**: Next.js deployed on Vercel for optimal frontend performance.

### Architectural Domains
| Domain | Primary Concern | Technology |
|--------|-----------------|------------|
| **Environment & Tooling** | Multi-environment orchestration | Vercel, Firebase CLI, Terraform |
| **Identity & Access** | Isolated authentication | Firebase Auth + Custom Claims |
| **Data Layer** | Schema-less isolated persistence | Firestore + Security Rules |
| **Evidence Management** | Secure, immutable audit evidence | Firebase Storage + Cloud Functions |
| **Business Logic** | Serverless task orchestration (DAG) | Cloud Functions (Node.js) |
| **Automated Comms** | Multi-channel notifications | Firebase Extensions (Email/SMS) |
| **UI/UX Foundation** | Responsive, modular frontend base | Next.js (Vercel), Tailwind CSS |
| **Real-time Sync** | Live exercise execution tracking | Firestore Real-time SDK |
| **Compliance Reporting** | Audit-ready PDF generation | Cloud Functions + Puppeteer |
