## Sub-Phase 1.1: Environment & Tooling
**Primary Concern**: Set up the foundational environment and CI/CD pipelines for GCP, Firebase, and Vercel.
**Dependencies**: Phase 0 complete.

**Atomic Task List**:
- [ ] Task 1.1.1: **Provision Firebase Project** and enable Firestore, Authentication, Storage, and Cloud Functions.
- [ ] Task 1.1.2: **Configure Vercel Integration** for the project repository with multi-environment support (Dev/Staging/Prod).
- [ ] Task 1.1.3: **Initialize Firebase CLI** in the repository and set up environment aliases.
- [ ] Task 1.1.4: **Develop GitHub Actions workflows** for automated deployments to Vercel and Firebase Functions.

---

## Sub-Phase 1.2: Identity & Access Management
**Primary Concern**: Implement a secure, tenant-isolated authentication system using Firebase Auth and Custom Claims.
**Dependencies**: Sub-Phase 1.1.

**Atomic Task List**:
- [ ] Task 1.2.1: **Configure Firebase Auth Providers** (Email/Password, Google) in the Firebase Console.
- [ ] Task 1.2.2: **Implement Custom Claims logic** in a Cloud Function to attach `tenantId` and `roles` to User ID Tokens.
- [ ] Task 1.2.3: **Develop Tenant Onboarding script** to programmatically create tenants and assign initial Admin users with Custom Claims.
- [ ] Task 1.2.4: **Verify Custom Claims propagation** by inspecting the JWT on the client side.

---

## Sub-Phase 1.3: Data Layer & Multi-Tenancy
**Primary Concern**: Design and implement a multi-tenant Firestore schema with robust Security Rules.
**Dependencies**: Sub-Phase 1.2.

**Atomic Task List**:
- [ ] Task 1.3.1: **Design Firestore Schema** for `tenants`, `exercises`, `tasks`, and `resources` collections.
- [ ] Task 1.3.2: **Implement Firestore Security Rules** to enforce `tenantId` isolation at the document level.
- [ ] Task 1.3.3: **Configure shared reference data collection** with public-read permissions.
- [ ] Task 1.3.4: **Write unit tests for Security Rules** using the Firebase Local Emulator Suite.

---

## Sub-Phase 1.4: Evidence Management
**Primary Concern**: Securely capture, hash, and store audit-ready evidence in Firebase Storage.
**Dependencies**: Sub-Phase 1.3.

**Atomic Task List**:
- [ ] Task 1.4.1: **Configure Firebase Storage buckets** and implement Security Rules for tenant-isolated uploads.
- [ ] Task 1.4.2: **Develop Cloud Function for Metadata Envelope** to automatically extract metadata and calculate SHA-256 hashes on upload.
- [ ] Task 1.4.3: **Implement Virus Scanning integration** via Cloud Function (using GCP Marketplace or external API).
- [ ] Task 1.4.4: **Enforce evidence immutability** in Storage Security Rules once the associated task is marked as `Completed`.

---

## Sub-Phase 1.5: Core Business Logic & Orchestration
**Primary Concern**: Implement the custom DAG orchestration engine and task state management using Cloud Functions.
**Dependencies**: Sub-Phase 1.3.

**Atomic Task List**:
- [ ] Task 1.5.1: **Develop DAG Validation Function** to prevent circular dependencies on task creation/update.
- [ ] Task 1.5.2: **Implement Task State Transition logic** (Cloud Function) triggered by Firestore updates to manage task progression.
- [ ] Task 1.5.3: **Calculate timing and variance metrics** server-side within Cloud Functions on task completion.
- [ ] Task 1.5.4: **Implement Go/No-Go decision logic** and audit logging for critical decision points.

---

## Sub-Phase 1.6: Automated Communications
**Primary Concern**: Configure and deploy automated email and SMS notification services via Firebase Extensions.
**Dependencies**: Sub-Phase 1.5.

**Atomic Task List**:
- [ ] Task 1.6.1: **Install "Trigger Email" Firebase Extension** and configure it with the project's email provider.
- [ ] Task 1.6.2: **Configure SMS notifications** via Twilio or GCP-native messaging services.
- [ ] Task 1.6.3: **Develop Notification Dispatcher Function** to trigger communications based on exercise events (e.g., Task Started, Milestone Delayed).
- [ ] Task 1.6.4: **Implement delivery status tracking** and log it back to the Firestore `audit_log` collection.
