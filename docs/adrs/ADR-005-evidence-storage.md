# ADR-005: Evidence Storage & Integrity (v4.0 - Modular GCP/Firebase MVP)

**Status:** Accepted  
**Date:** 2026-03-31
**Deciders:** Primary Agent, Security Agent, DevOps Agent  
**Technical Context:** Resilience Exercise Evidence Collection - Single-Module MVP
**Compliance Impact:** SOC 2 CC7.2, ISO 27001 A.12.2.1

---

## Context

The platform requires a secure, SOC 2 / ISO 27001 compliant evidence storage solution for Resilience exercises with support for:
- Evidence collection: Photo (25MB max) and Text only (MVP scope)
- Metadata Envelope: {who, when_utc, where_plan_id, sha256_hash}
- Evidence immutability post-completion
- Virus scanning (Photos only)
- Direct-to-Storage presigned URLs (for scalability)
- **Free-Tier Priority**: Optimize for storage and egress costs.

Previous model was Firebase Storage. Shifting to **Firebase / Google Cloud Storage**.

---

## Decision

Use **Firebase / Google Cloud Storage with Firebase Storage Security Rules** and **Cloud Functions for Virus Scanning**:

### Evidence Upload Pipeline (MVP Scope)
- **Direct Upload**: Client-side Firebase Storage SDK (using Security Rules to enforce `tenantId` isolation and file limits).
- **Metadata Envelope**: Automatically generated by Cloud Functions (`onFinalize` trigger) and stored in the `tasks` collection (within the `evidence` field).
- **SHA-256 Hash**: Calculated by Cloud Functions post-upload and verified for integrity.
- **Virus Scanning**: Cloud Functions trigger for virus scanning (using a GCP Marketplace solution or a third-party API like Cloudmersive).
- **Immutability**: Firebase Storage Security Rules will prevent overwriting or deleting files once the task is marked `Completed`.

### Benefits
- **Cost**: Generous free tier for Google Cloud Storage and Firebase Cloud Functions.
- **Integration**: Native integration with Firebase Auth and Security Rules.
- **Scalability**: No proxy server required for uploads; direct-to-storage is standard.

### Verification
- [ ] Evidence upload functional with photo (25MB limit) and text.
- [ ] SHA-256 hash verification confirms evidence integrity post-upload.
- [ ] Virus scan quarantine workflow tested with malicious file sample.
- [ ] Evidence immutability after task completion verified by Security Rules.
